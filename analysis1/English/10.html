Applications in Computer Science

1. Sequences and Convergence: The Essence of Iterative Algorithms ⚙️

The study of sequences, as in Exercise 1, is the mathematical foundation of iterative algorithms. In computer science, many problems are solved not in a single step, but by refining a solution step-by-step. Each iteration produces a new value (a term of the sequence) that is hopefully closer to the final solution (the limit).

Practical Example: Google's PageRank

Google's original algorithm, PageRank, works this way. It assigns an importance score to each web page. The score of a page at step n+1 is calculated based on the scores of all pages linking to it at step n. It's a massive recursive sequence! The algorithm stops when the scores barely change, meaning the sequence has converged.

// Initialize the scores of all pages to 1/N
    scores = initializeScores(numPages);
    
    // Loop until convergence
    while (not converged) {
      new_scores = calculateNextScores(scores, webGraph);
      
      // Check if the scores have stopped changing significantly
      if (distance(new_scores, scores) < epsilon) {
        converged = true;
      }
      
      scores = new_scores; // u_{n+1} = f(u_n)
    }
    return scores;

2. Continuity and Derivatives: The Engine of AI 🧠

The concepts of continuity and differentiability (Exercise 2) are absolutely central to artificial intelligence and Machine Learning. To train an AI model, a "cost function" is defined to measure how wrong its predictions are. The goal is to minimize this cost.

Practical Example: Gradient Descent

The most common method for minimizing this cost is "gradient descent." It relies on calculating the derivative (the gradient) of the cost function. The derivative indicates the direction of the steepest ascent. To reduce the error, the algorithm adjusts the model's parameters in the direction opposite to the derivative. This is only possible if the cost function is differentiable, and therefore continuous, at least piecewise.

// Model parameters (weights)
    weights = random_initialization();
    learning_rate = 0.01;
    
    // Training loop
    for (epoch in 1..1000) {
      // Calculate how wrong the prediction is
      error = calculateError(data, weights);
      
      // Calculate the derivative of the error with respect to the weights
      // f'(x)
      gradient = calculateGradient(error, weights);
      
      // Adjust weights in the direction that minimizes the error
      // x_new = x_old - learning_rate * f'(x)
      weights -= learning_rate * gradient;
    }

3. Existence Theorems: Proving that an Algorithm Works 💡

Rolle's Theorem (Exercise 3) and the Mean Value Theorem (MVT, Exercise 4) are "existence theorems." They don't give us the solution, but they guarantee that a solution exists. In theoretical computer science, these tools are essential for proving the correctness and stability of numerical algorithms.

Practical Example: Numerical Error Analysis

When a computer performs a calculation, it uses approximations (e.g., floating-point numbers). The MVT is used to bound the error introduced by these approximations. It allows us to say: "I don't know the exact error, but I can guarantee (thanks to the MVT) that it will never exceed a certain value M \cdot |x-a|." This guarantee is crucial for critical applications like aerospace simulations or financial calculations, where an uncontrolled error could be catastrophic.

// Concept of error analysis with the MVT
    // We want to approximate f(x) near 'a'
    // MVT says: f(x) - f(a) = f'(c) * (x-a) for some 'c' between a and x
    
    function approximate(f, x, a) {
      // Our approximation (e.g., a straight line)
      approximation = f(a); 
      
      // Real error = f(x) - approximation = f(x) - f(a)
      real_error = f(x) - f(a);
    
      // We don't know 'c', but we can find the maximum value of f'(x)
      // on the interval [a, x]
      M = find_max(derivative(f), interval=[a, x]);
    
      // We can guarantee that the error is bounded
      // |real_error| <= M * |x-a|
      assert(abs(real_error) <= M * abs(x-a));
      
      return approximation;
    }
